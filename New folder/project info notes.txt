While Pandas is the "gold standard" for tabular data (like Excel sheets or CSVs), it isn't used here for a few specific reasons:

Binary Format: The data is stored in an EDF (European Data Format) file. This is a complex binary format, not a text-based table. mne is a specialized library designed to read these files while keeping the metadata (like channel names and sampling rates) intact. 

Memory Efficiency: EEG data can be massive (thousands of samples per second). mne handles this efficiently by allowing us to "lazy load" data, whereas Pandas would try to load every single number into a table, which could crash your memory.

Signal Processing Tools: mne comes with built-in tools for filtering and artifacts removal that Pandas lacks.

2. Why did we write a separate model to combine it all?
This is a fantastic question. In the earlier steps, we were writing "scratchpad" code. We were testing the CWT to see if it looked right, and then we built a tiny dummy CNN just to see if the tensor shapes fit together.

Think of it like building a car:

Steps 1-3: We were building the engine parts on a workbench and testing them individually to make sure they sparked.

Step 4 (model.py): We bolted the engine, transmission, and wheels into a single chassis.

In PyTorch, to actually train an AI, every mathematical step must be contained inside a single nn.Module class. When you pass data into that combined model, PyTorch secretly builds a "computation graph." It tracks every single multiplication and addition. Later, when we tell the model it made a wrong guess, PyTorch uses that graph to trace its steps backward (Backpropagation) and fix its own math. If the CNN and the Transformer were in separate scripts, PyTorch couldn't connect the dots to learn!
